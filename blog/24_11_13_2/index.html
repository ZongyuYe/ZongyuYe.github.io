Towards Efficient Satellite Computing Through Adaptive Compression

https://ieeexplore.ieee.org/stamp/stamp.jsp?tp=&arnumber=10697471.

This is a paper for clarify data on the satellite and then compress it to improve the overall efficiency.

C. Yang et al., “Towards Efficient Satellite Computing Through Adaptive Compression,” in IEEE Transactions on Services Computing, doi: 10.1109/TSC.2024.3470341.

Two Main Problems:

  1. The response time from dispatching commands to satellites to obtaining detailed image analysis on the ground typically exceeds 8 hours (even a day).
  2. Performing processes directly in orbit is frequently deemed impractical due to the insufficient accuracy of the obtained onboard results.

(Limited onboard computing capability, Insufficient satellite-ground transmission volume)

In this case, this paper want to use image compression method to compress image data and then transmit them back to the ground. This also brings two problems.

  1. Since then downlink bottleneck, we can only transmitting useful data back to the ground. However, uniform compression to all images significantly degrades system accuracy. Thus, how to balance compression and transmission volume by determining the optimal image compression ratio is key to optimizing the capability of transmission.
  2. To improve the accuracy of the satellite computing system, downloaded images require restoring more information with less time overhead. However, restoring the resolution of all compressed images introduces substantial time overhead.

For restoration, we can only restore the most important part in an image.

Conidence-adaptive Image Compression

This paper utilizes compression techniques to prioritize and transmit vital image regions based on perceived transmission capacity and improves the inference accuracy using ground-based models.

$D$ is a set of image which need be downloaded.

$D_i$ is specific image.

$f_i$ represents the loss of information for each image after compression.

$C$ represents the compression factors, and $C_k$ represents the compression value applied to a specific image.

where $t$ is the duration of connectivity between the satellite and the ground station, $B$ denotes the downlink bandwidth, and $ξ$ represents the total storage size occupied by all processed images.

$R_{ij}$ represents the $j$-th region of the $i$-th image.

Then, the set of original images $D$ can be redefined as $R = {R_{11}, · · · ,R_{1j}, · · · ,R_{i1}, · · · , R_{ij}}, i ∈ I, j ∈ J .$

Moreover, based on the confidence values from swift annotated regions, we sort the elements within each R in descending order, denoted as $R_S= {R_{S_{11}}, · · · ,R_{S_{1j}}, · · · ,R_{S_{i1}}, · · · , R_{S_{ij}}}, i ∈ I, j ∈ J .$

We also sorted the elements within C in descending order, defined as $C_S= {C_{S_1},C_{S_2}, · · · ,C_{S_k}}.$

Moreover, we defined $T = C_SR_Sx$, where $T_{ij}$ represents that we use $C_{S_k} ∈ C_S$ to compress the $j$-th region of the $i$-th image.

$T_{ij} = C_{S_1}R_{S_{ij}}x_{i1} + C_{S_2}R_{S_{ij}}x_{i2} + · · · + C_{S_k}R_{S_{ij}}x_{ij} .$
